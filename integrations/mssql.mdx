This article guides you through the process of connecting MSSQL to Peaka, allowing you to seamlessly access your MSSQL data within the Peaka platform.

## Prerequisites

Before you start, you will need the following information from MSSQL.

- Host (Endpoint without the port or database name)
- Port (The port used to connect to your database)
- User (The username of the service account)
- Password (The password for the service account user)
- Database Name (The database name of your service)
- SSL, if active (Toggle "Connect Through TLS" switch)

## Connecting MSSQL to Peaka

1. Go to the settings page on Peaka.

2. On the Settings page, click "Add First Connection."

3. In the modal, select "MSSQL" from the list of available integrations.

4. Fill out the connection details form as shown in the image below, including:

   - Host (Endpoint without the port or database name)
   - Port (The port used to connect to your database)
   - User (The username of the service account)
   - Password (The password for the service account user)
   - Database Name (The database name of your service)
   - Connect Through TLS (Toggle switch if SSL is active)

5. Click "Next" to establish the connection.

<img src="https://cdn.peaka.com/integrations/mssql-1.png" />

Congratulations! You've successfully created a connection between MSSQL and Peaka. You can now access and manage your MSSQL data seamlessly within the Peaka platform.
