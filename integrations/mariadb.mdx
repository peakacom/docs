This article guides you through the process of connecting MariaDB to Peaka, allowing you to seamlessly access your MariaDB data within the Peaka platform.

## Prerequisites

Before you start, you will need the following information from MariaDB.

- Host (Endpoint without the port or database name)
- Port (The port used to connect to your database)
- User (The username of the service account)
- Password (The password for the service account user)
- SSL, if active (Toggle "Connect Through TLS" switch)

## Connecting MariaDB to Peaka

1. Go to the settings page on Peaka.

2. On the Settings page, click "Add First Connection."

3. In the modal, select "MariaDB" from the list of available integrations.

4. Fill out the data source details form as shown in the image below, including:

   - Host (Endpoint without the port or database name)
   - Port (The port used to connect to your database)
   - User (The username of the service account)
   - Password (The password for the service account user)
   - Connect Through TLS (Toggle switch if SSL is active)

5. Click "Next" to establish the data source.

<img src="https://cdn.peaka.com/integrations/mariadb-1.png" />

Congratulations! You've successfully created a new data source between MariaDB and Peaka. You can now access and manage your MariaDB data seamlessly within the Peaka platform.
