This article guides you through the process of connecting SAP HANA to Peaka, allowing you to seamlessly access your SAP HANA data within the Peaka platform.

## Prerequisites

Before you start, you will need the following information from SAP HANA.

- Host (endpoint without the port or database name)
- Port (the port used to connect to your database)
- User (the username of the service account)
- Password (the password for the service account user)
- Database Name (the database name of your service)
- SSL, if active (toggle "Connect through TLS" switch)
- Show System Schemas (enable the switch to display style schemas)

## Connecting SAP HANA to Peaka

1. Go to the Settings page on Peaka.

2. On the Settings page, click "Add First Connection."

3. In the modal, select "SAP Hana" from the list of available integrations.

4. Fill out the connection details form as shown in the image below, including:

   - Host (endpoint without the port or database name)
   - Port (the port used to connect to your database)
   - User (the username of the service account)
   - Password (the password for the service account user)
   - Database Name (the database name of your service)
   - Connect Through TLS (toggle switch if SSL is active)
   - Show System Schemas (enable its switch to display style schemas)

5. Click "Next" to establish the connection.

<img src="https://cdn.peaka.com/integrations/saphana-1.png" />

Congratulations! You've successfully created a connection between SAP HANA and Peaka. You can now access and manage your SAP HANA data seamlessly within the Peaka platform.
