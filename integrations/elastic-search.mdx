This article guides you through the process of connecting ElasticSearch to Peaka, allowing you to seamlessly access your ElasticSearch data within the Peaka platform.

## Prerequisites

Before you start, you will need the following information from ElasticSearch.

- Host
- Port
- User name
- SSL (Enable this option if you want to use SSL)

Follow the steps below to obtain it:

1. To obtain the Host and Port, navigate to the developer page and copy the ElasticSearch cluster URL. You can extract the Host and Port from the URL.

<img src="https://cdn.peaka.com/integrations/elastic-search-1.png" />

2. Navigate to the developer page and click on the "Reset Password" button as shown in the image.

<img src="https://cdn.peaka.com/integrations/elastic-search-2.png" />

3. To obtain User name and Password, cLick on the Reset Password button as in the image, and your default user name is "elastic".

<img src="https://cdn.peaka.com/integrations/elastic-search-3.png" />

## Connecting ElasticSearch to Peaka

Now that you have obtained the necessary integration data and credentials, you can connect ElasticSearch to Peaka using the following steps:

1. Navigate to the data model page in Peaka.

2. Click the "Bring your data" button to initiate the connection process.

3. In the modal that appears, select "ElasticSearch" as your connection type.

4. Enter a name for your connection, input the credentials you obtained in the previous steps.

<img src="https://cdn.peaka.com/integrations/elastic-search-4.png" />

5. Click "Authorize" to create your ElasticSearch connection.

Congratulations! You've successfully connected ElasticSearch to Peaka. You can now access and manage your ElasticSearch data seamlessly within the Peaka platform.
